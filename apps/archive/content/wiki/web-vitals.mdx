## 탄생 배경

- Google에서 개발한 프로젝트로서, 웹사이트의 성능 개선을 촉진하기 위해 만들어졌다.
- 사용자 경험을 개선하려면 페이지 로딩 속도, 상호작용 대기 시간 등을 측정하여 개발자에게 통찰력을 제공하는 것이 목표

## 구성

### Largest Contentful Paint (LCP)

- 페이지의 주요 컨텐츠가 로드되는 시간을 측정
- 사용자가 가장 중요하게 여기는 내용이 얼마나 빨리 나타나는지 확인하는 지표

### First Input Delay (FID)

- 사용자가 페이지와 상호작용하려고 할 때까지의 대기 시간을 측정합
- 페이지가 얼마나 반응적인지를 보여주는 지표

### Cumulative Layout Shift (CLS)

- 페이지 로딩 중에 레이아웃이 얼마나 안정적으로 유지되는지 측정
- 사용자가 예기치 않게 클릭한 위치가 바뀌는 것을 방지하기 위한 지표

## 할 수 있는 것과 이점

- 웹사이트의 성능을 모니터링하고 측정
- 문제가 발생하는 부분을 식별하고 최적화
- 사용자 경험을 개선하기 위해 성능 개선 작업 수행

### 이점

- `사용자 경험 개선`: 좋은 Web Vitals 점수를 유지하면 사용자들이 더 나은 경험을 느낄 수 있습니다.
- `검색 엔진 최적화 (SEO)`: Google은 Web Vitals를 검색 순위에 고려합니다. 따라서 높은 점수를 유지하면 검색 결과에서 더 높은 순위를 기대할 수 있습니다.
- `고객 유지`: 성능이 좋은 웹사이트는 사용자들이 더 오래 머무르고 더 많은 작업을 수행하게 만들어 고객 유지에 도움을 줍니다.
